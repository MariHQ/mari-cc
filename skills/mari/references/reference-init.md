# init — project editorial setup

Run `mari init style` from the repository root. The setup creates or updates the editorial context and hook configuration while preserving existing project prose.

After setup:

1. Read `PRODUCT.md` and `STYLE.md`.
2. Confirm the base style guide in `.mari/config.json`.
3. Run `mari detect` on a representative document.
4. Review any project-specific terminology with the user before adding glossary entries.
